faceless-explainer 工作流复盘

本文用于复盘这次“玉伯 / YouMind”视频制作：标准流程本来如何设计、每一步依赖什么、产出什么、质量门检查什么，以及这次为什么“流程完整”但成片仍然没有达到高质量发布级。

一、标准流程的核心定位

它适合做什么

faceless-explainer 的输入是文章、笔记、主题或 brief；输出是无真人出镜的解释型视频。

画面不依赖网站截图或真实产品界面，而是用文字排版、抽象图形、图解、数据可视化来解释内容。

它不适合做什么

它不是产品宣传片流程，不是网站录屏流程，也不是把已有视频重新包装的流程。

如果用户给的是产品站、PR、访谈视频或已有音频，应该走其他 skill，而不是强塞进 faceless-explainer。

原话：Input is arbitrary text. Output is a faceless explainer video: no captured website, no product screenshots — every visual is invented by the LLM.

三、这次实际选择复盘

下面不是标准流程说明，而是这次“玉伯 / YouMind”项目在每一步实际做过的选择。它能解释为什么我们虽然跑通了流程，但最后仍然出现了“流程完整、创意质量不够”的问题。

步骤 当时可选项 实际选择 选择原因 后果 / 复盘
路由判断 faceless-explainer / product-launch-video / general-video 选择 faceless-explainer 用户明确要求“按照 faceless-explainer 标准产线”，输入也更像一篇人物 / 商业认知文章，而不是产品官网宣传。 方向基本成立。但文章里有 YouMind 产品信息，容易滑向产品片；后续视觉如果不加约束，就会变成泛泛的商业解释视频。
素材获取 用 Chrome 抓微信文章 / 用户粘贴全文 / 只用摘要 最终使用用户粘贴的全文 Chrome 获取微信文章不稳定；用户直接给了完整文本，信息完整且可控。 文本源可靠，但失去了原文排版、图片、标题层级等上下文。faceless 路线本来不依赖真实截图，所以可以接受。
Step 0.0 brief 重新确认主题、角度、时长、画幅、语言 / 按默认推进 按默认推进：中文、横屏、解释型短片 用户说“按默认来”，并要求尽快按标准流程做成视频。 效率高，但创作 brief 不够硬：没有提前锁定“受众是谁、片子要打中的情绪是什么、发布平台是什么”。这是质量隐患。
Step 0 init 复用已有项目 / 初始化新项目 新建 videos/yubo-demystification-explainer/ 这是一个新主题，且要独立验证 faceless-explainer 流程。 目录清晰，产物完整；后续复盘也比较容易。
Step 1 scaffold 合成 faceless capture 包 / 做网页截图资产包 合成 visible-text.txt 和 synthetic tokens faceless-explainer 明确要求不抓网站、不用产品截图。 符合标准流程。代价是画面只能靠抽象图形和文字表达，不能借真实产品界面增强具体感。
Step 2 scriptwriting 强钩子传播稿 / 忠实文章结构 / 产品宣传稿 选择“故事解释 + 概念解释”：story-explainer with concept-explainer，9 个 scene 原文是一篇人物和创业认知文章，最自然的结构是沿着“祛魅、产品观、PMF、体积、飞轮、诚实”展开。 准确性较好，但传播钩子偏弱。标准 scriptwriting 能生成顺稿，不保证爆款开头。
Step 2 风格选择 block-frame / capsule / claude / pin-and-paper / scatterbrain 选择 pin-and-paper 文本气质偏克制、诚实、反包装，纸张笔记风比较贴合。 风格匹配，但也容易变成“黄纸 + 线框 + 标签”的模板感。后续需要 style frame 审批来打破惯性。
Step 3 audio 标准 audio.mjs 分 scene TTS / Seed Audio 单条全片音频 按用户要求使用 Seed Audio 生成单条全片音频 用户明确要求“千万不要分段生成”，并希望 Seed Audio 一次性生成完整旁白、背景氛围和节奏。 声音连贯性更好，但偏离标准音频链路。为兼容 HyperFrames，又补了 seed-audio-full-track.mjs、inject-full-track-audio.mjs 等脚本。
音频转写 HyperFrames transcription / 本地 Whisper / 不做 transcript 最终用本地 Python Whisper 转写并归一化 HyperFrames CLI transcription 路径遇到本机 whisper 兼容性问题；不能没有 transcript，因为字幕和节奏需要它。 功能上补齐了 transcript，但这是一次工程绕路。复盘上说明：音频改成 Seed Audio 后，transcript 适配必须成为正式步骤。
Step 4 visual-design 忠实文章段落图解 / 更强 MG 隐喻 / 产品界面化表达 选择 faceless 抽象图解：气泡、仪表、版本尺、阶梯、分数卡、PMF 气球、体积几何、飞轮 这最符合 faceless-explainer 的“invented visuals”要求，也避免真实产品截图。 结构完整，但画面偏“解释图表”，不是足够 motion-first 的 MG。视觉没有提前经过 style frame 和 storyboard 审批。
Step 5 prep 每 scene 独立 / 连续场景分组 分成 6 个 worker：1-2、4-5、8-9 合并，其余独立；总时长约 75 秒 连续场景共享同一视觉对象，有利于减少割裂感。 工程分组合理。但如果前置视觉设计不够强，分组只能保证连续，不能自动保证好看。
Step 5.5 captions 标准 caption 脚本 / 自定义中文适配 / 不加字幕 保留字幕，并做中文 transcript / caption 适配 中文 explainer 没字幕会明显降低可看性；但标准 grouper 对中文不如英文自然。 字幕可用，但这类中文项目应该把“中文断句、字幕节奏、字幕行长”当成正式质量门，而不是临时补丁。
Step 6 scenes 严格复用设计系统 / 每场自由发挥 / 引入外部素材 严格复用设计系统，不引入真实产品截图和人物图片 符合 faceless-explainer 标准，也能减少版权和素材依赖。 工程稳定，但画面同质化。faceless 不等于只能用模板图形；它仍然需要导演级视觉策略。
Step 7 preflight 只跑机器门 / 加人工导演审片 机器门 + contact sheet 复查 这是标准流程里的 finalize 方式：先修机器发现的问题，再做一次 lean contact-sheet look。 能保证技术可交付，但不能充分保证传播性、审美、信息节奏。我们后面发现的问题，多数属于机器门看不出来的问题。
最终输出 一次交付 / 继续做质量版 / 回到前置重做 产出多个版本，最后保留 video-faceless-quality.mp4 第一次成片质量不足，所以继续做了质量修正和 review contact sheet。 最终比初版好，但仍暴露出一个根本问题：如果前面没有 brief、旁白、style frame、storyboard 审批，后面修成片会很被动。

五、标准流程内置的质量保障

机器质量门

这些门能保证视频工程上不坏：不缺文件、不炸时间线、不遮字幕、不明显重叠、不溢出、能渲染。

创意质量门

标准流程里只有轻量创意检查：scriptwriting、visual-design、scene worker、finalize 的一次 contact sheet。

这能避免明显坏片，但不能保证片子高级、好看、传播性强。

七、复盘结论

faceless-explainer 是一个自动成片工程管线。 它擅长把文本变成结构完整、可渲染、技术门通过的视频。

但它不是完整的导演级创意制作流程。 它不强制做旁白审稿、style frame 审批、storyboard 审批、动态分镜审稿、逐句视听对齐、最终导演审片。

所以这次成片的问题不是“流程没跑”，而是“标准流程本身的创意质量保障偏轻”。

九、Visual Design 重做记录

这次重做没有重新生成旁白，而是从 Step 4 visual-design 开始重走后续流程： 保留单条 Seed Audio 全片音频和 transcript 字幕，替换视觉设计与 HyperFrames 实现。

新方向叫“诚实工作台 / X-ray notebook”：用暖纸背景、黑色结构线和一条红色主线贯穿全片。 目标是让画面像一个连续的 explainer 世界，而不是多个 worker 各自生成的卡片拼贴。

环节 这次选择 产物 当前判断
视觉方向 单一视觉世界，红色主线贯穿 9 个 beat section_plan.creative-v2.md 方向比旧版统一，但仍是第一轮 style frame 级别
HyperFrames 实现 单个 composition 替代多 worker 场景拼接 #composition-files-excluded 减少风格断裂，便于统一调色和节奏
主入口 index.html 只挂载 creative_v2 + 原音频 + 字幕 index.html 可在 HyperFrames Studio 直接预览新版

自动质量门结果：lint 0 error / validate 0 error / inspect 0 layout issue。

人工审稿判断：新版已经解决了“风格各自为政”的问题，但还没有达到发布级。 主要问题是视觉主体仍偏克制，像线稿演示稿，缺少更强的封面级关键帧和更丰富的 motion texture。

下一步如果继续提升，应该不是再盲目生成一版，而是先定 2-3 张关键 style frame： 开场“祛魅”、中段“好看优先”、结尾“诚实产品”。这三张立住后，再扩展整片。
